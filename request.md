

=== 구조 분석 ===
**📌 프로젝트 유형: 모바일 앱**

이 모바일 앱은 15개의 독립 뷰로 구성되어 있다.


**독립 뷰 1: '플로깅-view'**
*   '플로깅-view'가 위치한다.
    -   기능: 플로깅 활동의 주요 화면을 구성하고 요소를 배치함.
    *   이 '플로깅-view'는 다시 4행 1열의 뷰로 구성되어 있다.
        *   1행에는 '플로깅 타이틀-text-field'가 위치한다.
            -   기능: "DIGG X GONGSIM"
        *   2행에는 '출력 영역(1)-view'가 위치한다.
            -   기능: "사진-저장 폴더"에 있는 사진을 가장 최근 사진 순으로 '카메라 기록-이미지/텍스트 표시'이와 같은 형식으로 보여줌
            *   이 '출력 영역(1)-view'는 다시 1개의 요소로 구성되어 있다.
                *   '카메라 기록-display'가 위치한다.
                    -   기능: 촬영된 이미지 또는 비디오를 표시하고 관리함.
                    *   이 '카메라 기록-display'는 다시 1개의 요소로 구성되어 있다.
                        *   '삭제-touch-button'가 위치한다.
                            -   기능: 카메라 기록 내 선택된 항목을 제거하도록 요청함.
        *   3행에는 '카메라 뷰-display'가 위치한다.
            -   기능: 실시간 카메라 영상을 출력하여 플로깅 활동을 지원함.
        *   4행에는 '버튼 그룹-view'가 위치한다.
            -   기능: 플로깅 화면 내 기능 버튼들을 묶어 레이아웃을 구성함.
            *   이 '버튼 그룹-view'는 다시 1행 3열의 뷰로 구성되어 있다.
                *   1열에는 '로그아웃-touch-button'가 위치한다.
                    -   기능: 현재 사용자 세션을 종료하고 로그인 페이지로 이동함.
                *   2열에는 '촬영 버튼-touch-button'가 위치한다.
                    -   기능: 카메라 뷰의 현재 화면을 이미지로 캡처하도록 지시함.
                *   3열에는 '대쉬보드-touch-button'가 위치한다.
                    -   기능: 사용자의 플로깅 활동 통계 화면으로 이동을 요청함.

=== 사용자 기능 흐름 ===
사용자가 앱에 처음 접속하면 **로그인-view** 화면이 표시되며, 상단에는 **앱 이름-display**가 나타납니다. 사용자는 **구글 로그인 버튼-touch-button**을 클릭하여 구글 계정으로 로그인하거나, **구글 계정 전환-touch-button**을 통해 다른 구글 계정으로 전환할 수 있습니다. 또한, **자동 로그인 체크-touch-button**을 클릭하여 **자동 로그인 타이틀-display**에 명시된 "자동 로그인" 기능을 활성화할 수 있습니다.

로그인 또는 앱 최초 실행 후, **안내-popup**이 나타나며, 여기에는 **플로깅 타이틀-안내용-display**와 함께 "본 앱을 활성화하기 위해서는 사용자의 위치와 카메라 기능을 요구 합니다"라는 내용의 **안내(1)-display**가 표시됩니다. 사용자는 **안내 승인 여부 버튼-view** 내의 **거절-touch-button** 또는 **승인-touch-button**을 클릭하여 권한 요청에 응답합니다. 만약 **거절-touch-button**을 클릭하면, **승인 여부-condition**의 "미 승인 시" 조건에 따라 다시 **로그인-view**로 돌아갑니다. 반면, **승인-touch-button**을 클릭하면 **승인 여부-condition**의 "승인 시" 조건에 따라 **플로깅-view** 화면으로 이동합니다.

**플로깅-view** 화면에서는 상단에 **플로깅 타이틀-text-field**가 표시되며, 중앙에는 **카메라 뷰-display**를 통해 실시간 카메라 화면이 나타납니다. 사용자는 **촬영 버튼-touch-button**을 클릭하여 사진을 촬영할 수 있습니다. 촬영된 사진은 **사진-folder**에 저장되며, 이어서 **API 호출-function** (Gemini API 호출)이 트리거되어 사진을 분석합니다. 분석된 사진 데이터(촬영 일자, 위도, 경도, 쓰레기 카테고리, 방치 시 오염 영향력)는 **사진 정보-data**에 저장됩니다. 동시에 **사진-folder**에 저장된 사진들은 **출력 영역(1)-view** 내의 **카메라 기록-display**에 가장 최근 사진 순으로 표시됩니다. 사용자는 **출력 영역(1)-view**에서 **삭제-touch-button**을 클릭하여 **사진 삭제-function**을 실행할 수 있으며, 이는 **사진-folder**와 **사진 정보-data**에서 해당 사진 정보를 삭제합니다.

**플로깅-view** 하단에는 **버튼 그룹-view**가 위치하며, 여기에는 **로그아웃-touch-button**과 **대쉬보드-touch-button**이 포함되어 있습니다. 사용자가 **대쉬보드-touch-button**을 클릭하면, **데이터 로드-function**이 실행되어 사용자 데이터를 로드하고, 이 데이터는 **대쉬보드-view** 화면에 표시됩니다.

**대쉬보드-view** 화면에서는 상단에 **플로깅 타이틀(1)-text-field**가 표시되고, **대쉬보드-display**는 맨 처음 로드 시 오늘 기준의 데이터를 보여줍니다. **플로깅한 날-view** 영역에서는 **활동한 루트 맵 및 줏은 쓰레기 유형 마크-display**를 통해 활동 경로와 쓰레기 유형이 시각화되며, **차트-view** 내의 **쓰레기 유형 차트-display**, **이동 거리-display**, **줏은 쓰레기 수-display**, **오염도 저감 수준-display**를 통해 상세 통계가 제공됩니다. **히스토리-view** 영역에는 **필터링-view**가 있어 사용자가 1주, 1달 등 기간별로 데이터를 필터링할 수 있으며, 필터링 결과는 **플로깅 대쉬보드-display**, **쓰레기 유형 차트(2)-display**, **쓰레기 유형 차트(1)-display**, **오염도 저감 수준(1)-display**에 연동되어 표시됩니다.

**대쉬보드-view** 하단에는 **버튼 그룹(1)-view**가 있으며, 여기에는 **메인 홈 돌아가기-touch-button**, **캘린더-touch-button**, **내보내기-touch-button**이 있습니다. **메인 홈 돌아가기-touch-button**을 클릭하면 **플로깅-view**로 돌아갑니다. **캘린더-touch-button**을 클릭하면 **캘린더-popup**이 열리며, 여기에는 **오늘 날짜-display**, **이전달-touch-button**, **다음달-touch-button**이 있고, **캘린더-view**는 쓰레기를 주운 날짜를 플로깅 아이콘으로 표시하고 해당 날짜의 쓰레기 수와 이동 거리를 보여줍니다. **캘린더-view**에서 날짜를 선택하면 **대쉬보드-display**의 데이터가 해당 날짜 기준으로 업데이트됩니다. 마지막으로, **내보내기-touch-button**을 클릭하면 **AI 분석-popup**이 열리며, **사진 정보-data**를 활용한 AI 분석 기능이 실행됩니다.

=== UI 요소 상호작용 ===
## UI 요소 상호작용 - 상세 분석

### 로그인 화면 (`로그인-view`)

**`구글 로그인 버튼-touch-button`을 클릭하면:**
현재 JSON 데이터에는 `구글 로그인 버튼-touch-button`에서 시작하는 직접적인 연결이 명시되어 있지 않습니다. 하지만 일반적인 시나리오를 고려할 때, 이 버튼을 클릭하면 구글 계정을 통한 인증 프로세스가 시작될 것으로 추정됩니다. 인증이 성공적으로 완료되면, 앱은 사용자에게 필요한 권한을 요청하는 `안내-popup`으로 이동하거나, 바로 `플로깅-view` 화면으로 전환될 수 있습니다.

**`자동 로그인 체크-touch-button`을 클릭하면:**
`자동 로그인 체크-touch-button`은 `자동 로그인 타이틀-display`와 함께 자동 로그인 기능을 활성화하거나 비활성화하는 토글 동작을 수행할 것으로 추정됩니다. 이 기능의 활성화 여부는 사용자 설정에 저장되어 다음 앱 실행 시 자동으로 로그인될 수 있도록 합니다. 현재 JSON에는 직접적인 연결이 명시되어 있지 않아 추가적인 UI 변화는 미정의입니다.

**`구글 계정 전환-touch-button`을 클릭하면:**
현재 JSON 데이터에는 `구글 계정 전환-touch-button`에서 시작하는 직접적인 연결이 명시되어 있지 않습니다. 이 버튼을 클릭하면 사용자가 현재 로그인된 구글 계정 외에 다른 구글 계정으로 전환하여 로그인할 수 있는 인터페이스(예: 계정 선택 팝업 또는 시스템 계정 관리 화면)가 표시될 것으로 추정됩니다.

### 안내 팝업 (`안내-popup`)

**`로그인-view`에서 `안내-popup`이 표시되면:**
앱 시작 또는 로그인 직후, `안내-popup`이 화면에 나타나 사용자에게 앱 활성화를 위해 위치 및 카메라 기능이 필요함을 알립니다. `플로깅 타이틀-안내용-display`와 `안내(1)-display`가 이 정보를 시각적으로 제공합니다. `안내 승인 여부 버튼-view` 내의 `거절-touch-button`과 `승인-touch-button`이 사용자 선택을 기다립니다.

**`승인-touch-button`을 클릭하면:**
`안내 승인 여부 버튼-view` 내의 `승인-touch-button`을 클릭하면, `안내 승인 여부 버튼-view`는 `승인 여부-condition`으로 연결됩니다. `승인 여부-condition`은 "승인 시" 조건에 따라 `플로깅-view` 화면으로 전환됩니다. 이 과정에서 앱은 사용자에게 위치 및 카메라 기능 사용 권한을 요청하고, 사용자가 이를 허용하면 `플로깅-view`가 활성화되어 플로깅 활동을 시작할 수 있게 됩니다.

**`거절-touch-button`을 클릭하면:**
`안내 승인 여부 버튼-view` 내의 `거절-touch-button`을 클릭하면, `안내 승인 여부 버튼-view`는 `승인 여부-condition`으로 연결됩니다. `승인 여부-condition`은 "미 승인 시" 조건에 따라 `로그인-view` 화면으로 다시 돌아갑니다. 이는 앱의 핵심 기능인 플로깅이 위치 및 카메라 권한 없이는 불가능하므로, 사용자에게 권한 허용을 다시 유도하기 위함으로 추정됩니다.

### 플로깅 화면 (`플로깅-view`)

**`촬영 버튼-touch-button`을 클릭하면:**
`플로깅-view` 화면에서 `카메라 뷰-display`를 통해 카메라 미리보기가 활성화된 상태에서 `촬영 버튼-touch-button`을 클릭하면 사진이 촬영됩니다. 촬영된 사진은 `사진-folder`에 저장되며, 동시에 `API 호출-function`을 통해 Gemini API로 전송되어 쓰레기 유형 등의 분석이 요청됩니다. 분석 결과는 `사진 정보-data`에 촬영 일자, 위도, 경도, 쓰레기 카테고리, 방치 시 오염 영향력 등의 상세 정보로 기록됩니다. 또한, `출력 영역(1)-view` 내의 `카메라 기록-display`에 가장 최근 촬영된 사진이 목록 형태로 표시됩니다.

**`삭제-touch-button`을 클릭하면:**
`출력 영역(1)-view` 내의 `카메라 기록-display`에 표시된 특정 사진 옆의 `삭제-touch-button`을 클릭하면, 해당 사진은 `사진 삭제-function`을 통해 `사진-folder`에서 파일이 삭제되고, `사진 정보-data`에서도 관련 데이터가 제거됩니다. 이후 `카메라 기록-display`는 업데이트된 사진 목록을 보여줍니다.

**`로그아웃-touch-button`을 클릭하면:**
현재 JSON 데이터에는 `로그아웃-touch-button`에서 시작하는 직접적인 연결이 명시되어 있지 않습니다. 이 버튼을 클릭하면 앱에서 사용자 세션이 종료되고, `로그인-view` 화면으로 전환될 것으로 추정됩니다.

**`대쉬보드-touch-button`을 클릭하면:**
`버튼 그룹-view` 내의 `대쉬보드-touch-button`을 클릭하면, `데이터 로드-function`이 호출됩니다. `데이터 로드-function`은 사용자의 플로깅 데이터를 대시보드에 표시하기 위한 내부 검증 로직을 수행한 후, `대쉬보드-view` 화면으로 전환됩니다.

### 대쉬보드 화면 (`대쉬보드-view`)

**`대쉬보드-view` 화면이 로드되면:**
`플로깅 타이틀(1)-text-field`에 "DIGG X GONGSIM"이라는 기본 타이틀이 표시됩니다. `대쉬보드-display`는 맨 처음 로드 시 오늘 날짜를 기준으로 사용자의 플로깅 활동 데이터를 표시합니다. `활동한 루트 맵 및 줏은 쓰레기 유형 마크-display`는 사용자가 활동한 루트 맵과 주운 쓰레기 유형을 시각적으로 보여줍니다. `차트-view` 내의 `쓰레기 유형 차트-display`, `이동 거리-display`, `줏은 쓰레기 수-display`, `오염도 저감 수준-display`는 해당 날짜의 플로깅 통계 데이터를 차트 형태로 제공합니다. `히스토리-view` 내의 `필터링-view`는 1주, 1달, 3달, 6달, 1년, 3년, 5년 단위로 데이터를 필터링할 수 있는 옵션을 제공하며, 선택된 기간에 따라 `플로깅 대쉬보드-display`, `오염도 저감 수준(1)-display`, `쓰레기 유형 차트(1)-display`, `쓰레기 유형 차트(2)-display`의 데이터가 연동되어 업데이트됩니다.

**`캘린더-touch-button`을 클릭하면:**
`버튼 그룹(1)-view` 내의 `캘린더-touch-button`을 클릭하면, `캘린더-popup`이 화면에 표시됩니다.

**`메인 홈 돌아가기-touch-button`을 클릭하면:**
현재 JSON 데이터에는 `메인 홈 돌아가기-touch-button`에서 시작하는 직접적인 연결이 명시되어 있지 않습니다. 이 버튼을 클릭하면 `플로깅-view` 화면으로 돌아갈 것으로 추정됩니다.

**`내보내기-touch-button`을 클릭하면:**
`버튼 그룹(1)-view` 내의 `내보내기-touch-button`을 클릭하면, `사진 정보-data`에 저장된 플로깅 데이터를 가져와 `AI 분석-popup`으로 연결됩니다. `AI 분석-popup`이 화면에 표시되며, AI 분석 기능을 수행하여 사용자에게 추가적인 통찰을 제공할 것으로 추정됩니다.

### 캘린더 팝업 (`캘린더-popup`)

**`캘린더-popup`이 로드되면:**
`오늘 날짜-display`에 현재 월과 연도가 표시됩니다. `캘린더-view`는 쓰레기를 주운 날짜에 플로깅 아이콘 이미지를 표시하고, 해당 날짜에 주운 쓰레기 수와 이동 거리를 일자별로 함께 표시합니다. `범례-view`는 캘린더에 사용된 아이콘이나 색상에 대한 설명을 제공하여 사용자가 데이터를 쉽게 이해할 수 있도록 돕습니다.

**`이전달-touch-button`을 클릭하면:**
`이전달-touch-button`을 클릭하면 `오늘 날짜-display`에 표시되는 월이 이전 달로 변경되고, `캘린더-view`의 내용도 이전 달의 플로깅 데이터로 업데이트됩니다.

**`다음달-touch-button`을 클릭하면:**
`다음달-touch-button`을 클릭하면 `오늘 날짜-display`에 표시되는 월이 다음 달로 변경되고, `캘린더-view`의 내용도 다음 달의 플로깅 데이터로 업데이트됩니다.

**`캘린더-view`에서 특정 날짜를 클릭하면:**
`캘린더-view`에서 특정 날짜를 클릭하면 `캘린더-popup`이 닫히고, `대쉬보드-display`에 해당 날짜의 플로깅 데이터가 로드되어 표시됩니다. 이는 사용자가 특정 날짜의 활동 기록을 상세히 확인하고자 할 때 유용합니다.

### AI 분석 팝업 (`AI 분석-popup`)

**`AI 분석-popup`이 로드되면:**
`내보내기-touch-button`을 통해 `AI 분석-popup`이 열리면, `사진 정보-data`에서 가져온 플로깅 데이터를 기반으로 AI 분석 결과를 표시할 것으로 추정됩니다. 현재 JSON에는 팝업 내부 요소에 대한 상세 정보가 미정의되어 있지만, 쓰레기 유형별 통계, 환경 영향 분석, 개선 제안 등 다양한 AI 기반 분석 결과를 제공할 것으로 예상됩니다.

=== 데이터 흐름 및 저장 ===
[입력 수집]
사용자는 앱 시작 시 `안내-popup`을 통해 앱 활성화에 필요한 위치 및 카메라 기능 요구사항을 `안내(1)-display`("본 앱을 활성화하기 위해서는 사용자의 위치와 카메라 기능을 요구 합니다")로 확인합니다. 이후 `안내 승인 여부 버튼-view` 내의 `승인-touch-button` 또는 `거절-touch-button`을 클릭하여 권한 요청에 대한 입력을 제공합니다. 권한 승인 후 `플로깅-view` 화면으로 진입하며, `플로깅 타이틀-text-field`에 "DIGG X GONGSIM"이 표시됩니다.

`플로깅-view`에서 사용자는 `촬영 버튼-touch-button`을 클릭하여 사진을 촬영합니다. `출력 영역(1)-view`에 표시된 `카메라 기록-display` 내의 사진을 `삭제-touch-button`을 클릭하여 삭제할 수 있습니다.

`버튼 그룹-view` 내의 `대쉬보드-touch-button`을 클릭하여 대쉬보드 화면으로 이동합니다. `버튼 그룹(1)-view` 내의 `캘린더-touch-button`을 클릭하여 `캘린더-popup`을 열고, `이전달-touch-button`과 `다음달-touch-button`을 통해 월을 변경하며 특정 날짜를 선택하여 데이터를 조회합니다. `필터링-view`에서는 "1주, 1달, 3달, 6달, 1년, 3년, 5년" 등의 옵션을 선택하여 `플로깅 대쉬보드-display`에 표시될 데이터의 기간을 조정합니다. 마지막으로 `내보내기-touch-button`을 클릭하여 AI 분석을 위한 데이터를 전송할 수 있습니다.

[데이터 가공]
`안내 승인 여부 버튼-view`에서 발생한 사용자 입력은 `승인 여부-condition`으로 전달되어, 사용자의 승인 여부에 따라 앱의 다음 흐름이 결정됩니다. `카메라 뷰-display`를 통해 캡처된 사진 데이터는 `사진-folder`에 저장됩니다.

`사진-folder`에 저장된 사진은 `API 호출-function` (Gemini API 호출)으로 전달되어 이미지 분석이 수행됩니다. 이 분석을 통해 사진의 메타데이터(촬영 일자, 위도, 경도, 쓰레기 카테고리, 방치 시 오염 영향력)가 추출됩니다.

`삭제-touch-button`이 트리거하는 `사진 삭제-function`은 `사진-folder`에서 해당 사진 파일을 삭제하고, `사진 정보-data`에서 해당 사진의 메타데이터를 제거합니다.

`대쉬보드-touch-button` 클릭 시 `데이터 로드-function`이 호출되어 `사진 정보-data`에서 사용자의 데이터를 로드하며, "사용자가 겹치지 않게 내부 검증 로직"을 구현하여 데이터를 정제합니다. `캘린더-popup` 내의 `캘린더-view`에서 날짜가 선택되면, 해당 날짜에 해당하는 `사진 정보-data`가 필터링되어 `대쉬보드-display`에 표시될 데이터로 가공됩니다. `필터링-view`에서 선택된 기간에 따라 `사진 정보-data`가 필터링되어 `플로깅 대쉬보드-display`에 연동될 데이터가 가공됩니다.

[API 전송]
`사진-folder`에 저장된 사진 데이터는 `API 호출-function` (Gemini API 호출)을 통해 외부 API로 전송되어 이미지 분석을 요청합니다. (구체적인 API 엔드포인트 및 요청 형식은 추가 명세 필요)

[응답 처리]
`API 호출-function` (Gemini API 호출)로부터 수신된 이미지 분석 결과(메타데이터)는 `사진 정보-data` (table-indexing)에 저장됩니다. 이 데이터는 "촬영 일자", "위도", "경도", "쓰레기 카테고리", "방치 시 오염 영향력" 필드를 포함합니다.

[저장/표시]
**저장**: 캡처된 사진 파일은 `사진-folder`에 저장됩니다. 이미지 분석을 통해 추출된 메타데이터는 `사진 정보-data` (table-indexing)에 정의된 스키마에 따라 저장됩니다.

**표시**:
*   `승인 여부-condition`의 결과에 따라, 권한 미승인 시 `로그인-view` 화면으로 돌아가고, 권한 승인 시 `플로깅-view` 화면으로 이동합니다.
*   `플로깅-view` 화면에서는 `플로깅 타이틀-text-field`에 "DIGG X GONGSIM"이 표시됩니다.
*   `출력 영역(1)-view`는 `사진-folder`에 있는 사진을 가장 최근 사진 순으로 `카메라 기록-display`에 표시합니다.
*   `대쉬보드-view`는 `데이터 로드-function`을 통해 `사진 정보-data`에서 가져온 데이터를 기반으로 `플로깅 타이틀(1)-text-field`("DIGG X GONGSIM")와 `대쉬보드-display`를 통해 플로깅 활동 데이터를 시각화하여 보여줍니다. `대쉬보드-display`는 "맨 처음 로드 시 오늘 기준이며 '캘린더-touch-button'로 캘린더에서 날짜 클릭 시 해당 날짜의 데이터 보여줌" 기능을 수행합니다.
*   `대쉬보드-display`는 `활동한 루트 맵 및 줏은 쓰레기 유형 마크-display` (활동 루트 맵 및 쓰레기 유형 마크), `차트-view` 내의 `쓰레기 유형 차트-display` (쓰레기 유형 차트), `이동 거리-display` (이동 거리), `줏은 쓰레기 수-display` (줏은 쓰레기 수), `오염도 저감 수준-display` (오염도 저감 수준) 등의 상세 정보를 표시합니다.
*   `캘린더-popup` 내의 `캘린더-view`는 `사진 정보-data`를 기반으로 쓰레기를 주운 날짜를 플로깅 아이콘 이미지로 표기하고, 해당 날짜에 대한 쓰레기 수와 이동 거리를 일자별로 표시합니다. `오늘 날짜-display`는 현재 날짜를 보여줍니다.
*   `히스토리-view` 내의 `플로깅 대쉬보드-display`는 `필터링-view`에서 선택된 기간에 따라 `사진 정보-data`를 연동하여 `쓰레기 유형 차트(2)-display`, `쓰레기 유형 차트(1)-display`, `오염도 저감 수준(1)-display` 등의 형태로 과거 플로깅 데이터를 표시합니다.
*   `내보내기-touch-button`을 통해 `사진 정보-data`가 전달된 `AI 분석-popup`은 추가적인 AI 분석 결과를 사용자에게 표시합니다.
